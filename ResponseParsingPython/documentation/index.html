What are leading zeros?

A 'leading zero' is any '0' digit that appears at the start of a number.

For example

0023 has two leading zeros
00023 has three leading zeros
23 has no leading zeros

All of the above are valid ways of writing the number 23. Most of the time we would never write '0023' or '00023', but all of the above technically represent the number 23. If the answer to a question were 23, but the student gives their answer as '0023', they are still correct.

There are two parameters for how to handle leading zeros

'Allow Leading Zeros'

'Remove Leading Zeros'
– can be true or false – can be true or false
– set to false by default – set to false by default

This parameter determines whether or not students can submit an answer that has leading zeros.

If it is set to false for a question, then if a student tries to submit an answer to that question like '0023', they will not be allowed to submit the answer, and will be given the message 'You don't need to write zeros at the start of your answer.'

If it is set to true, then a student can submit answers like '023', '0023', '00023', and all will be allowed through.

Imagine that a student enters '0023' as their answer to a question. (And that 'Allow Leading Zeros' has been set to true for that question.)

If 'Remove Leading Zeros' is set to true, then this answer will be converted into '23' before it is compared to the key.

This means that even if a student does put leading zeros in their answer, it doesn't matter, because we're going to remove them before checking their answer.

If this is set to false, leading zeros won't be removed from the student's answer before it is checked.

Common settings

The most common set of values that you are likely to want to use for the above two parameters are

'Allow Leading Zeros' true
'Remove Leading Zeros' true

This is the "friendliest" or "nicest" setting. A student can type leading zeros in their answer (if they really want to), but they will automatically be removed before their answer is checked. This way, if a student types '0023', but the key is '23', they won't be "told off" and they won't get the question wrong.

Sometimes, however, you might want to encourage the student not to write leading zeros. (This would be particularly useful at lower g-values, because we want to encourage young students to write numbers in the conventional way.) In this case, you could set 'Allow Leading Zeros' to false - then if they try to type '0023', they will be told not to write the leading zeros.

What are trailing zeros?

A 'trailing zero' is any '0' digit that appears at the end of a number and after the decimal point.

For example

1.500 has two trailing zeros
1.5000 has three trailing zeros
5.0000 has four trailing zeros

But note that

1.00500 has only two trailing zeros
200.40 has only one trailing zero
200.0040 has only one trailing zero
23000 has no trailing zeros

When numbers are considered to be exact, by convention we don't normally write out any trailing zeros to a number (because the trailing zeros technically go on forever).

There are two parameters for how to handle trailing zeros

... and they're very similar to the parameters for leading zeros.

'Allow Trailing Zeros'

'Remove Trailing Zeros'
– can be true or false – can be true or false
– set to true by default – set to false by default

This parameter determines whether or not students can submit an answer that has trailing zeros.

If it is set to false for a question, then if a student tries to submit an answer to that question like '2.500', they will not be allowed to submit the answer, and will be given the message 'You don't need to write zeros at the end of your answer.'

If it is set to true, then a student can submit answers like '2.50', '2.500', '2.5000', and all will be allowed through.

Imagine that a student enters '2.500' as their answer to a question. (And that 'Allow Trailing Zeros' has been set to true for that question.)

If 'Remove Trailing Zeros' is set to true, then this answer will be converted into '2.5' before it is compared to the key.

This means that even if a student does put trailing zeros in their answer, it doesn't matter, because we're going to remove them before checking their answer.

If this is set to false, trailing zeros won't be removed from the student's answer before it is checked.

An exception ...

Neither of the parameters 'Allow Trailing Zeros' or 'Remove Trailing Zeros' can be set for the currency value primary type. If you try to set them, they will be ignored.

This is because there are often conventions for how currency values are written. For example, in the UK, it's considered incorrect to write '£5.5' - the correct way to write this is '£5.50'.

For currency value questions, the key should always be written with two decimal places or no decimal places, and the student will always be told to write their answer to either two decimal places or no decimal places.

Common settings

The most common set of values that you are likely to want to use for the above two parameters are

'Allow Trailing Zeros' true
'Remove Trailing Zeros' true

This is the "friendliest" or "nicest" setting. A student can type trailing zeros in their answer (if they really want to), but they will automatically be removed before their answer is checked. This way, if a student types '2.500', but the key is '2.5', they won't be "told off" and they won't get the question wrong.

Plus and minus signs

When we write negative numbers, we always write a minus sign at the start of the number to indicate that it's negative.

When we write positive numbers, however, by convention we usually don't write a plus sign at the start of the number to indicate that it's positive. If a number is written without a plus or a minus sign, it's taken to be positive.

However, writing a positive number with a plus sign is still a valid way of writing a number, even if it's not how we normally write numbers.

Sometimes we may want to encourage a student not to write a plus sign at the start of a positive number (because we want to encourage them to follow the conventional rules for writing numbers). Other times, we may want them to explicitly write the plus sign (for example, when giving relative charges of particles in physics). And sometimes we may want the allow the student to do either, and have their answer marked correct regardless of whether or not it has the plus sign.
